## Phase Details

### Phase 05: UI Polish
**Goal**: Polish the public Astro site UI so reading and navigation feel more intentional and brand-aligned while preserving existing accessibility/privacy/analytics behavior and static-site constraints.
**Depends on**: Phase 04
**Requirements**: UIP-01, UIP-02, UIP-03 (completed)
**Success Criteria** (what must be TRUE):
  1. Long-form pages apply the intended polish (visual system and justified long-form presentation) while keeping locked readability/accessibility baselines.
  2. Every synthesis article includes the required editor profile block fields and standard contact sentence near the top of the page.
  3. Homepage/navigation reflect phase IA changes: desktop research-theme image map with mobile fallback links, `/overview` added, and `/featured` removed.
**Plans**: 3 plans
