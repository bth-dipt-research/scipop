## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User multiselect input → DataFrame filtering | User selections may result in empty DataFrame or invalid column references |
| Column selection → Topic model input | Invalid or empty column selections could cause downstream failures |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-05 | Denial of Service | Empty filtered DataFrame | mitigate | Validate filtered DataFrame length before proceeding; disable "Next" button if empty; show clear error message |
| T-06-06 | Information Disclosure | Column name exposure in UI | accept | Column names are from user-uploaded data; no additional info leaked |
| T-06-07 | Tampering | Invalid column selection (empty or non-existent columns) | mitigate | Validate column selection is non-empty; filter out index artifacts from options; check columns exist in DataFrame |
| T-06-08 | Denial of Service | Fingerprint computation on very large filtered DataFrame | accept | Local tool, user controls dataset size; fingerprint computation is O(n log n) from sorting, acceptable for typical datasets (<100k rows) |
</threat_model>
