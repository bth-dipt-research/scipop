# Phase 2: Readability & Accessibility Baseline - Context

**Gathered:** 2026-04-03
**Status:** Ready for planning

<domain>
## Phase Boundary

Ensure public synthesis content is readable and navigable on desktop and mobile, and include required disclaimer treatment on synthesis narrative destinations.

</domain>

<decisions>
## Implementation Decisions

### Disclaimer coverage and content model
- **D-01:** Disclaimer is required on all synthesis narrative pages: `/syntheses/{slug}` and `/featured`.
- **D-02:** Disclaimer is not required on `/methodology` for this phase.
- **D-03:** Use a shared default disclaimer text with optional per-page override support.
- **D-04:** Disclaimer appears as a final section at the end of each required article.
- **D-05:** If a required page has no disclaimer available, fail the build with a clear error.

### Disclaimer presentation and copy
- **D-06:** Render disclaimer as a subtle callout block (visually distinct but non-alarmist).
- **D-07:** Use neutral plain-language tone for general audiences.
- **D-08:** Standardize heading as `Important context`.
- **D-09:** Default disclaimer copy is:
  "This text was generated by AI and edited by humans. It is based on SERL's research publications between January 2020 and September 2025. For technical questions, please contact michael.unterkalmsteiner@bth.se."

### Reading layout baseline
- **D-10:** Target desktop body line length to a 65-75ch reading measure.
- **D-11:** Body text scales from `16px` on mobile to `18px` on desktop.
- **D-12:** Paragraph line-height target is `1.65`, with tighter heading line-height.
- **D-13:** Apply a consistent vertical spacing scale for headings, paragraphs, and section blocks.

### Navigation behavior across devices
- **D-14:** On mobile, primary navigation uses a compact menu button pattern.
- **D-15:** On desktop, keep visible horizontal top navigation links.
- **D-16:** Navigation baseline must include keyboard operability, visible focus states, and minimum 44px touch targets.
- **D-17:** Mobile menu auto-closes after a link is selected.

### the agent's Discretion
- Exact CSS token values and breakpoint cutoffs that satisfy the locked typography/spacing decisions.
- Exact interaction details (animation timing, iconography) for the mobile menu, as long as keyboard/focus/touch baselines are met.

</decisions>

## Existing Code Insights

### Reusable Assets
- `site/src/layouts/BaseLayout.astro`: shared header/navigation wrapper used by public routes; primary integration point for nav and global typography styles.
- `site/src/pages/syntheses/[slug].astro`: synthesis article rendering path where disclaimer section must be injected.
- `site/src/pages/featured/index.astro`: featured synthesis article rendering path where disclaimer section must be injected.
- `site/src/content.config.ts`: content-collection schema surface for adding shared/override disclaimer metadata if needed.

### Established Patterns
- Public routes are Astro pages using shared `BaseLayout` and content collections.
- Route-level content guards already use build-time failures when required content is missing (throwing explicit errors).
- Synthesis detail pages are generated with `getStaticPaths` from approved content entries.

### Integration Points
- Add disclaimer data fields in content schema/frontmatter and consume them in synthesis + featured pages.
- Add shared responsive reading styles in `BaseLayout` so all public pages inherit baseline typography and spacing.
- Implement mobile navigation toggle behavior in `BaseLayout` while preserving current desktop navigation visibility.
